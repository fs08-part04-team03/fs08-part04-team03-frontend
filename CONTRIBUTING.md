# 기여 가이드

이 프로젝트에 기여해주셔서 감사합니다! 🎉

## 개발 프로세스

1. 이슈 생성 또는 기존 이슈 선택
2. 브랜치 생성 (`feature/이슈번호-기능명`)
3. 코드 작성 및 커밋
4. Pull Request 생성
5. 코드 리뷰
6. 머지

## 커밋 규칙

커밋 메시지는 다음 형식을 따릅니다:

```
<타입>: <제목> (#이슈번호)

[부가 설명]
```

자세한 내용은 [공통 컨벤션 문서](CONVENTIONS.md)를 참고하세요.

## 코드 리뷰

- 모든 PR은 최소 1명의 승인이 필요합니다
- 리뷰어는 24시간 이내 응답합니다
- 건설적인 피드백을 제공해주세요

## 질문하기

궁금한 점이 있다면:

- Discord 개발 채널에 질문
- GitHub Discussions 활용
- 이슈로 질문 등록

## 행동 강령

- 서로 존중하며 협력합니다
- 건설적인 피드백을 제공합니다
- 다양성을 존중합니다
